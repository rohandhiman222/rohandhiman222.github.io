# Node.js Concepts and Examples

# 1. Event-Driven Architecture

Explanation: Event-Driven Architecture in Node.js means that the system reacts to events as they occur. This is managed through the EventEmitter class in the events module. An event emitter allows one part of the code to emit an event, and another part to respond to it, enabling decoupled and asynchronous communication between different parts of the application.

Extended Example:

const EventEmitter = require("events");

class MyEmitter extends EventEmitter {}

const myEmitter = new MyEmitter();

// Registering an event listener
myEmitter.on("event", (a, b) => {
  console.log("An event occurred!", a, b);
});

// Emitting the event with arguments
myEmitter.emit("event", "arg1", "arg2");

// Another example with error handling
myEmitter.on("error", (err) => {
  console.error("An error occurred:", err);
});

// Emitting an error event
myEmitter.emit("error", new Error("Oops! Something went wrong."));

# 2. Non-Blocking I/O

Explanation: Non-blocking I/O allows Node.js to perform I/O operations asynchronously. This means that the code execution doesn't pause to wait for the I/O operations (like reading a file or making a network request) to complete. Instead, the operations are handled in the background, and the main thread can continue executing other code. When the I/O operation completes, a callback is invoked to handle the result.

Extended Example:

const fs = require("fs");

// Non-blocking read using a callback
fs.readFile("input.txt", "utf8", (err, data) => {
  if (err) {
    return console.error("Error reading file:", err);
  }
  console.log("File content:", data);
});

console.log("This message is logged before file reading completes");

// Non-blocking network request using a library like Axios
const axios = require("axios");

axios
  .get("https://jsonplaceholder.typicode.com/todos/1")
  .then((response) => {
    console.log("API response:", response.data);
  })
  .catch((error) => {
    console.error("API error:", error);
  });

console.log("This message is logged before API response is received");

# 3. Single-Threaded with Worker Threads

Explanation: Node.js operates on a single-threaded event loop model for executing JavaScript code. However, it uses an internal thread pool managed by libuv to handle asynchronous operations like I/O. For CPU-intensive tasks, which might block the event loop, Node.js provides worker threads to enable parallel processing.

Extended Example:

const { Worker, isMainThread, parentPort } = require("worker_threads");

if (isMainThread) {
  // This is the main thread
  const worker = new Worker(__filename);

  worker.on("message", (message) => {
    console.log("Received from worker:", message);
  });

  worker.postMessage("Hello, worker!");

  // Handling worker errors
  worker.on("error", (error) => {
    console.error("Worker error:", error);
  });

  // Handling worker exit
  worker.on("exit", (code) => {
    if (code !== 0) {
      console.error(`Worker stopped with exit code ${code}`);
    }
  });
} else {
  // This is the worker thread
  parentPort.on("message", (message) => {
    console.log("Received from main thread:", message);
    parentPort.postMessage("Hello, main thread!");
  });
}

# 4. Modules and NPM

Explanation: Modules in Node.js are reusable blocks of code that can be exported from one file and imported into another. This modularity helps in organizing code and promoting reusability. Node.js uses the CommonJS module system (require and module.exports). NPM is the default package manager for Node.js, providing access to thousands of reusable libraries and tools.

Extended Example:

// myModule.js
const sayHello = (name) => `Hello, ${name}!`;

module.exports = {
  sayHello,
  anotherFunction: () => "This is another function",
};

// main.js
const myModule = require("./myModule");

console.log(myModule.sayHello("World"));
console.log(myModule.anotherFunction());

Using NPM:

# Initializing a new Node.js project
npm init -y

# Installing a package
npm install express

# main.js with express
const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello, World!');
});

app.listen(3000, () => {
  console.log('Server is running on port 3000');
});

# 5. Streams

Explanation: Streams in Node.js are instances of EventEmitter and provide a way to handle streaming data. They are particularly useful for working with large files or incoming data from a network, where loading the entire data into memory would be inefficient.

Streams can be:

  • Readable: Stream from which data can be read.
  • Writable: Stream to which data can be written.
  • Duplex: Stream that is both readable and writable.
  • Transform: Duplex stream where the output is computed based on input.

Extended Example:

const fs = require("fs");

// Creating a readable stream
const readStream = fs.createReadStream("input.txt", "utf8");

// Creating a writable stream
const writeStream = fs.createWriteStream("output.txt");

// Piping data from the readable stream to the writable stream
readStream.pipe(writeStream);

readStream.on("data", (chunk) => {
  console.log("Reading chunk:", chunk);
});

writeStream.on("finish", () => {
  console.log("Writing finished");
});

# 6. Buffers and Binary Data

Explanation: Buffers are a way of handling binary data directly in Node.js. They are useful for reading and writing binary data (e.g., files, network protocols). Buffers are raw memory allocations that can represent a fixed-size chunk of memory outside the V8 heap, which is used for handling binary data.

Extended Example:

const buffer = Buffer.from("Hello, world!", "utf8");

console.log("Buffer:", buffer);
console.log("Buffer length:", buffer.length);
console.log("Buffer to string:", buffer.toString("utf8"));

// Working with binary data
const buffer1 = Buffer.alloc(10); // Creates a buffer of 10 bytes
buffer1.write("ABC", 0, "utf8");

console.log("Buffer1:", buffer1);
console.log("Buffer1 to string:", buffer1.toString("utf8"));

const buffer2 = Buffer.from([0x01, 0x02, 0x03, 0x04]);
console.log("Buffer2:", buffer2);

# 7. Promises and Async/Await

Explanation: Promises represent a value that may be available now, or in the future, or never. They are used to handle asynchronous operations in a more readable and maintainable way compared to callbacks. The async and await keywords provide a way to write asynchronous code that looks synchronous, making it easier to read and understand.

Extended Example:

// Using Promises
const doSomething = () => {
  return new Promise((resolve, reject) => {
    setTimeout(() => resolve("Done!"), 1000);
  });
};

doSomething()
  .then((result) => {
    console.log("Promise result:", result);
  })
  .catch((error) => {
    console.error("Promise error:", error);
  });

// Using async/await
const doSomethingAsync = async () => {
  try {
    const result = await doSomething();
    console.log("Async/await result:", result);
  } catch (error) {
    console.error("Async/await error:", error);
  }
};

doSomethingAsync();

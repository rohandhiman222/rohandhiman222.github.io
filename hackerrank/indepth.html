# Var

  1. Scope

var (Function-scoped)

  • Variables declared with var are scoped to the nearest function block. If declared outside any function, they are global.

  • var does not recognize block-level scope (like inside if or for blocks).

let and const (Block-scoped)

  • Variables declared with let and const are scoped to the nearest enclosing block, which can be a function, a loop, or a conditional block.

  • This means they are only accessible within that specific block.

// Var
function varScope() {
  var x = 1;
  if (true) {
    var x = 2; // same `x` as above
    console.log(x); // 2
  }
  console.log(x); // 2 (changed by the block)
}

varScope();

// Let const

function letConstScope() {
  let y = 1;
  if (true) {
    let y = 2; // different `y`
    console.log(y); // 2
  }
  console.log(y); // 1 (unaffected by the block)

  const z = 3;
  if (true) {
    const z = 4; // different `z`
    console.log(z); // 4
  }
  console.log(z); // 3 (unaffected by the block)
}

letConstScope();
  1. Hoisting

var (Hoisted and initialized with undefined)

  • Variables declared with var are hoisted to the top of their function scope.

  • This means the declaration is moved to the top, but the initialization remains in place.

  • As a result, they are initialized with undefined before any code is executed.

function varHoisting() {
  console.log(a); // undefined (hoisted)
  var a = 10;
  console.log(a); // 10
}

varHoisting();

  • The variable a is hoisted to the top of the function.

  • Before the line var a = 10; is executed, a is undefined.

  • After initialization, a becomes 10.

let and const (Hoisted but not initialized - Temporal Dead Zone)

  • Variables declared with let and const are hoisted to the top of their block.

  • Unlike var, they are not initialized until their declaration is encountered in the code.

  • Accessing them before the declaration results in a ReferenceError (this period is called the Temporal Dead Zone).

function letConstHoisting() {
  // console.log(b); // ReferenceError (temporal dead zone)
  let b = 10;
  console.log(b); // 10

  // console.log(c); // ReferenceError (temporal dead zone)
  const c = 20;
  console.log(c); // 20
}

letConstHoisting();
  1. Redeclaration

var (Can be redeclared)

  • Variables declared with var can be redeclared within the same scope without any error.

  • This can lead to unintended behavior and bugs.

function varRedeclaration() {
  var d = 1;
  var d = 2; // redeclaration allowed
  console.log(d); // 2
}

varRedeclaration();

  • The variable d is declared twice with var in the same scope.

  • The second declaration overwrites the first, resulting in d being

let and const (Cannot be redeclared)

  • Variables declared with let and const cannot be redeclared within the same scope.

  • Attempting to do so results in a syntax error.

function letConstRedeclaration() {
  let e = 1;
  // let e = 2; // SyntaxError: Identifier 'e' has already been declared

  const f = 3;
  // const f = 4; // SyntaxError: Identifier 'f' has already been declared
}

letConstRedeclaration();
  1. Reassignment

var and let (Can be reassigned)

  • Variables declared with var and let can be reassigned to a different value after their initial declaration.

  • This allows for mutable variables.

function varLetReassignment() {
  var g = 1;
  g = 2; // reassignment allowed
  console.log(g); // 2

  let h = 3;
  h = 4; // reassignment allowed
  console.log(h); // 4
}

varLetReassignment();

const (Cannot be reassigned, but object properties can be modified)

  • Variables declared with const cannot be reassigned after their initial declaration.

  • However, if the variable is an object or an array, its properties or elements can be modified.

function constReassignment() {
  const i = 5;
  // i = 6; // TypeError: Assignment to constant variable

  const obj = { key: "value" };
  obj.key = "new value"; // allowed
  console.log(obj.key); // 'new value'

  // obj = {}; // TypeError: Assignment to constant variable
}

constReassignment();

  • Attempting to reassign i results in a TypeError.

  • Modifying the properties of obj is allowed, but reassigning obj to a new object is not.

Summary

Scope:

  • var: Function-scoped or globally scoped.

  • let and const: Block-scoped. Hoisting:

  • var: Hoisted and initialized with undefined.

  • let and const: Hoisted but not initialized (temporal dead zone). Redeclaration:

  • var: Can be redeclared in the same scope.

  • let and const: Cannot be redeclared in the same scope. Reassignment:

  • var and let: Can be reassigned.

  • const: Cannot be reassigned, but properties of objects or arrays can be modified.

# Closures in JavaScript

A closure is a function that has access to its own scope, the scope of the outer function, and the global scope. Closures allow a function to access variables from an enclosing scope, even after that scope has finished executing.

Key Characteristics of Closures:

  1. Access to Outer Function’s Variables:
  • A closure can access variables from its own scope, the outer function’s scope, and the global scope.
  1. Preservation of Data:
  • Closures allow data to persist in memory even after the outer function has executed.
  1. Private Variables:
  • Closures can be used to create private variables, which are not accessible from outside the function.

Example 1: Basic Closure

function outerFunction() {
  let outerVariable = "I am from the outer function";

  function innerFunction() {
    console.log(outerVariable); // Accessing the outer variable
  }

  return innerFunction;
}

const closureExample = outerFunction();
closureExample(); // Output: 'I am from the outer function'

Example 2: Closure with Parameters

function createAdder(x) {
  return function (y) {
    return x + y; // Accessing parameter `x` from the outer function
  };
}

const add5 = createAdder(5);
console.log(add5(2)); // Output: 7
console.log(add5(10)); // Output: 15
  • createAdder is a function that takes a parameter x and returns an inner function.
  • The inner function takes a parameter y and returns the sum of x and y.
  • The inner function forms a closure that keeps the value of x from createAdder.

Example 3: Data Privacy with Closures

function Counter() {
  let count = 0;

  return {
    increment: function () {
      count++;
      console.log(count);
    },
    decrement: function () {
      count--;
      console.log(count);
    },
    getCount: function () {
      return count;
    },
  };
}

const counter = Counter();
counter.increment(); // Output: 1
counter.increment(); // Output: 2
counter.decrement(); // Output: 1
console.log(counter.getCount()); // Output: 1

  • Counter function maintains a private count variable.

  • It returns an object with methods to manipulate count.

  • These methods form closures that have access to the count variable, keeping it private.

Example 5: Immediate Invoked Function Expression (IIFE)

const counter = (function () {
  let privateCounter = 0;

  function changeBy(val) {
    privateCounter += val;
  }

  return {
    increment: function () {
      changeBy(1);
      console.log(privateCounter);
    },
    decrement: function () {
      changeBy(-1);
      console.log(privateCounter);
    },
    value: function () {
      return privateCounter;
    },
  };
})();

counter.increment(); // Output: 1
counter.increment(); // Output: 2
counter.decrement(); // Output: 1
console.log(counter.value()); // Output: 1

  • The counter is implemented using an IIFE.

  • The IIFE creates a private privateCounter and exposes methods to manipulate it.

  • The methods form closures over privateCounter, allowing controlled access.

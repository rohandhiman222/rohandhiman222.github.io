# Next.js Rendering Methods

Next.js offers several ways to render content, each with its own use cases and advantages. The four main rendering methods in Next.js are:

  1. Static Generation (SSG)
  2. Server-Side Rendering (SSR)
  3. Client-Side Rendering (CSR)
  4. Incremental Static Regeneration (ISR)

# 1. Static Generation (SSG)

Overview:

  • Static Generation generates HTML at build time.
  • Suitable for pages that can be pre-rendered ahead of a user's request.

Use Cases:

  • Marketing pages
  • Blog posts
  • Documentation
  • Product listings

Advantages:

  • Fast performance: Pre-rendered HTML is served from a CDN.
  • Improved SEO: Pages are fully rendered when crawled by search engines.
  • Scalability: Reduced load on the server since the content is served statically.

Example Implementation:

Create a new page at pages/static.js:

import fs from "fs";
import path from "path";

export async function getStaticProps() {
  const filePath = path.join(process.cwd(), "data.json");
  const jsonData = fs.readFileSync(filePath, "utf-8");
  const data = JSON.parse(jsonData);

  return {
    props: {
      data,
    },
  };
}

const StaticPage = ({ data }) => {
  return (
    <div>
      <h1>Static Generation Example</h1>
      <ul>
        {data.items.map((item) => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
    </div>
  );
};

export default StaticPage;

Detailed Steps:

  1. getStaticProps is called at build time.
  2. Fetch or read data, e.g., from a file or API.
  3. Return the data as props to the page component.
  4. The page is pre-rendered as HTML at build time and served statically.

# 2. Server-Side Rendering (SSR)

Overview:

  • Server-Side Rendering generates HTML on each request.
  • Suitable for pages that need to fetch data at request time or require user-specific content.

Use Cases:

  • Personalized dashboards
  • Dynamic content that changes per request
  • User authentication/authorization-dependent content

Advantages:

  • Fresh data on each request: Content is always up-to-date.
  • SEO benefits: Fully rendered HTML is sent to search engines.

Example Implementation:

Create a new page at pages/server.js:

export async function getServerSideProps() {
  const res = await fetch("https://api.example.com/data");
  const data = await res.json();

  return {
    props: {
      data,
    },
  };
}

const ServerPage = ({ data }) => {
  return (
    <div>
      <h1>Server-Side Rendering Example</h1>
      <ul>
        {data.items.map((item) => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
    </div>
  );
};

export default ServerPage;

Detailed Steps:

  1. getServerSideProps is called on every request.
  2. Fetch data, typically from an API.
  3. Return the data as props to the page component.
  4. The page is rendered on the server and sent to the client as HTML.

# 3. Client-Side Rendering (CSR)

Overview:

  • Client-Side Rendering fetches data and renders it entirely on the client side.
  • Suitable for pages that don’t need to be pre-rendered.

Use Cases:

  • User dashboards where data changes frequently
  • Pages with heavy interactions or animations
  • Single Page Applications (SPAs)

Advantages:

  • Fast initial load for static parts of the application.
  • Reduced server load: All rendering happens on the client.
  • Rich client-side interactivity.

Example Implementation:

Create a new page at pages/client.js:

import { useEffect, useState } from "react";

const ClientPage = () => {
  const [data, setData] = useState(null);

  useEffect(() => {
    fetch("https://api.example.com/data")
      .then((res) => res.json())
      .then((data) => setData(data));
  }, []);

  if (!data) return <div>Loading...</div>;

  return (
    <div>
      <h1>Client-Side Rendering Example</h1>
      <ul>
        {data.items.map((item) => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
    </div>
  );
};

export default ClientPage;

Detailed Steps:

  1. Use React's useEffect to fetch data after the initial render.
  2. Update the state with the fetched data.
  3. Render the component based on the updated state.

# 4. Incremental Static Regeneration (ISR)

Overview:

  • Incremental Static Regeneration allows you to update static content after you’ve built your application.
  • Generates static pages on the server as needed, allowing updates without a full rebuild.

Use Cases:

  • E-commerce product pages
  • News articles
  • Any page that needs to be updated frequently but can tolerate slight staleness

Advantages:

  • Combines the benefits of SSG and SSR.
  • Improved performance: Pages are statically generated but can be updated in the background.
  • Better scalability: Only the required pages are revalidated.

Example Implementation:

Create a new page at pages/isr.js:

import fs from "fs";
import path from "path";

export async function getStaticProps() {
  const filePath = path.join(process.cwd(), "data.json");
  const jsonData = fs.readFileSync(filePath, "utf-8");
  const data = JSON.parse(jsonData);

  return {
    props: {
      data,
    },
    revalidate: 10, // In seconds
  };
}

const ISRPage = ({ data }) => {
  return (
    <div>
      <h1>Incremental Static Regeneration Example</h1>
      <ul>
        {data.items.map((item) => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
    </div>
  );
};

export default ISRPage;

Detailed Steps:

  1. getStaticProps runs at build time like SSG.
  2. Define a revalidate period, e.g., 10 seconds.
  3. After the initial build, the page will be regenerated in the background on the first request after the revalidation period.

# Summary

# Static Generation (SSG):

  • Pros: Fast performance, SEO-friendly, low server load.
  • Cons: Content can become stale, requires rebuild for updates.

# Server-Side Rendering (SSR):

  • Pros: Always up-to-date content, good for dynamic data.
  • Cons: Higher server load, slower performance compared to SSG.

# Client-Side Rendering (CSR):

  • Pros: Rich interactivity, reduced server load.
  • Cons: Initial load can be slow, less SEO-friendly.

# Incremental Static Regeneration (ISR):

  • Pros: Combines benefits of SSG and SSR, improved performance and scalability.
  • Cons: Content can be slightly stale depending on revalidation period.

Choosing the right rendering method depends on your specific use case, performance requirements, and how often your content needs to be updated.

# Computed Properties

  • Computed properties allow us to define a property that is used the same way as data, but can also have some custom logic that is cached based on its dependencies. you can consider computed properties another view into your data.

    <h2>{{firstname}} {{lastname}}</h2>
    <h3>{{fullName}}</h3>

    <button @click="change">Click me</button>

  computed: {
    // using cache good for permormance

      set(newValue) {
        const split = newValue.split(" ");
        this.firstname = split[0];
        this.lastname = split[1];

      get() {
        return `${this.firstname} ${this.lastname}`;
      }
    }
  },
  methods: {
    change() {
      //   this.firstname = "hacker";

<template>

    <br />
    <span :class="validateLength ? 'maxLengthError':''">{{characterAvailable}}</span>

    <textarea type="text" v-model.trim="newTwit"></textarea>

    <button :disabled="!validateLength || disableButton" @click="addTweet">Sumit</button>

    <p class="maxLengthError" v-if="validateLength">{{ msgError }}</p>
    <br />
    <h2>Twitter Lists</h2>
    <div class="listTweet" v-for="(tweet, idx) in tweets" :key="idx">{{ tweet }}</div>
  </div>

<script>
export default {
  name: "TwitterChatDoneValidationComputed",
  data() {
    return {
      newTwit: "",
      maxLength: 140,
      msgError: "Max length is 140 please edit the twits",
      tweets: []
    };
  },
  methods: {
    addTweet() {
      if (this.newTwit === "") return;
      this.tweets.push(this.newTwit);
      this.newTwit = "";
    }
  },
  computed: {
    characterAvailable() {
      return this.maxLength - this.newTwit.length;
    },
    validateLength() {
      return this.newTwit.length > this.maxLength;
    },
    disableButton() {
      return this.newTwit.length === 0;
    }
  }
};
</script>

# Watchers

  • Watchers allow us to add functionality each time something changes, or respond to a particular change, we could watch a property and apply some logic. Watchers name matches what we are watching
<template>
  <div>
    <h2>{{firstname}}</h2>
    <button @click="change">Click me</button>
  </div>
</template>


<script>
export default {
  name: "TwitterChat",
  props: {
    msg: String
  },
  data() {
    return {
      firstname: "rohan"
    };
  },

  methods: {
    change() {
      this.firstname = "rohan sadhana";
    }
  },
  watch: {
    // track any data value by watacher
    firstname(newValue, oldValue) {
      console.log(`new value ${newValue} old value ${oldValue}`);
    }
  }
};
</script>

# lazy loading

TIP

When you visit this page that time load that page we called it lazy loading

this function help us lazy loading

component: () => import(/* webpackChunkName: "home" */ "../views/Home.vue"),

Router folder inside index.js

import Vue from "vue";
import VueRouter from "vue-router";
// import Home from "../views/Home.vue";
import Profile from "../views/Profile.vue";
import Profile2 from "../views/Profile2.vue";
import PageNotFound from "../views/PageNotFound.vue";
import Info from "../views/Info.vue";

Vue.use(VueRouter);

const routes = [
  {
    path: "/",
    name: "Home",
    // this is lazy loading when you visit this page when load this component
    component: () => import(/* webpackChunkName: "home" */ "../views/Home.vue"),
  },
  {
    path: "/profile/:username",
    name: "ProfileMain",
    component: Profile,
  },
  {
    path: "/profile/:username",
    name: "Profile",
    component: Profile,
    children: [
      // profile/username/info
      {
        path: "info",
        component: Info,
        name: "Info",
      },
    ],
  },

  {
    path: "/profile2",
    name: "Profile2",
    component: Profile2,
  },
  {
    path: "/about",
    name: "About",
    // route level code-splitting
    // this generates a separate chunk (about.[hash].js) for this route
    // which is lazy-loaded when the route is visited.
    component: () =>
      import(/* webpackChunkName: "about" */ "../views/About.vue"),
  },
  {
    path: "*",
    component: PageNotFound,
  },
];

const router = new VueRouter({
  routes,
});

export default router;

# Slots

  • Easy way to pass information into a component

  • Named slots allow you to specify exactly where the data is inserted

  • Scoped slots allow you to pass data from you child component back to the parent component

  • Allows you to insert data from a parent component into a child component

  • Different types of Slots

    • Named Slots
    • Scoped Slots
    • Dynamic Slot Names

# Optional chaining

Optional chaining allows reading the vlaue of a property located deep within a chain of connected object without having to check each reference in the chain

let obj = {
  name: "rohan",
};

console.log(obj?.address?.city); // undefined

# Nullish coalescing

The nullish coalescing operator (??) is a logical operator that returns its right hand operand when its left-hand operand is null or undefined, and otherwise returns its left-hand operand. this is a way to handle default values more predictably than using the or (||) operator.

const obj = {
  name: "rohan",
};

// this || work only falsy value also work null, undefined
const city = obj?.address?.city || "not found"; // work with falsy value

console.log(false || "false value"); //false value
console.log(0 || "false value"); // false value
console.log(null || "null value here"); // null value here
console.log(undefined || "undefined value here"); // undefined value here

// this ?? work only undefined or null otherwise return first value
// best use of ?? user?.age ===0 than print zero

const city2 = obj?.address?.city ?? "not found";
console.log(0 ?? "false value"); // 0
console.log(false ?? "false value"); // false
console.log(null ?? "null value"); // null value
console.log(undefined ?? "null value"); // undefined value

console.log(city); // not found
console.log(city); // not found

# Array.prototype.at()

const colors = ["red", "yello", "green"];
console.log(colors[0]); // red
console.log(colors[1]); // yellow
console.log(colors.at(0)); // red
console.log(colors.at(1)); // yello

// colors[0] colors.at(0) both are same only diff .at() support negative number

console.log(colors.at(-1)); // green

# .replaceAll() method for strings

const str = "lorem dog lore heeho good bye dog tha nice dog ok thanks dog";

let newvalue = str.replace("dog", "cat"); // replace only first value

let allnewValue = str.replace(new RegExp("dog", "g"), "cat"); // replace all

let allLastMethod = str.replaceAll("dog", "cat");
let allLastMethod2 = str.replaceAll(new RegExp("dog", "gi"), "cat");

console.log(newvalue); // lorem cat lore heeho good bye dog tha nice dog ok thanks dog
console.log(allnewValue); // lorem cat lore heeho good bye cat tha nice cat ok thanks cat
console.log(allLastMethod); // lorem cat lore heeho good bye cat tha nice cat ok thanks cat

// capital later also replace
console.log(allLastMethod2); // lorem cat lore heeho good bye cat tha nice cat ok thanks cat

# Logical assignment

Logical assignments are a combination of logical operators (&&, ||, ??) and assignment expressions.

let x = null;
let y = "default";

// Or assingment (x || = y) is equivalent to x || (x = y)
x ||= y;

console.log(x); // output 'default'

let a = 0;
let b = 2;

// And assignment (a &&= b) is equivalent to a && (a = b);

a &&= b;
console.log(a); // output 0

const todo = {
  priority: "",
  task: "finished editing",
};

// todo.priority ||= "Medium";
// todo.priority || (todo.priority = "Medium") // same above
// console.log(todo); { priority: 'Medium', task: 'finished editing' }

const todo2 = {
  priority: "test value",
  task: "finished editing",
};

todo2.priority &&= "Medium"; // priority should be true after that update value
// todo2.priority && (todo2.priority = "medium"); same above

console.log(todo2); // { priority: 'Medium', task: 'finished editing' }

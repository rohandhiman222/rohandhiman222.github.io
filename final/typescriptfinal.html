  • 1. What is TypeScript and how does it differ from JavaScript?

    # 1. What is TypeScript and how does it differ from JavaScript?

    Answer: TypeScript is a strongly typed superset of JavaScript that compiles to plain JavaScript. It adds static typing, classes, interfaces, and other features that help developers write more robust and maintainable code.

    Differences from JavaScript:

    • Static Typing: TypeScript supports static typing, which helps catch errors at compile time rather than runtime.
    • Interfaces: TypeScript introduces interfaces, which allow you to define the shape of an object.
    • Classes: TypeScript supports modern object-oriented programming features such as classes, inheritance, and access modifiers.
    • Generics: TypeScript supports generics, which enable you to write reusable and flexible code components.

    # 2. What are the benefits of using TypeScript?

    Answer:

    • Early Error Detection: TypeScript catches type-related errors at compile time, reducing runtime errors.
    • Improved Code Quality: Static typing and other TypeScript features lead to more robust and maintainable code.
    • Better IDE Support: TypeScript provides enhanced code completion, navigation, and refactoring in modern IDEs.
    • Readability and Maintainability: Type annotations and interfaces make the code more readable and easier to understand.
    • Cross-Platform: TypeScript can be used for both client-side and server-side development.

    # 3. How do you define a type for a variable in TypeScript?

    Answer: You define a type for a variable using a colon followed by the type after the variable name.

    Example:

    let age: number = 25;
let name: string = "John";
let isActive: boolean = true;

    # 4. What are interfaces in TypeScript and how do you use them?

    Answer: Interfaces in TypeScript define the shape of an object, specifying the types of properties that the object can have.

    Usage Example:

    interface Person {
  name: string;
  age: number;
}

const person: Person = {
  name: "John",
  age: 30,
};

    # 5. What are generics in TypeScript and how do you use them?

    Answer: Generics allow you to create reusable components that can work with different types. They provide a way to pass types as parameters to functions, classes, or interfaces.

    Usage Example:

    function identity<T>(value: T): T {
  return value;
}

let result = identity<number>(10); // Output: 10
let result2 = identity<string>("Hello"); // Output: 'Hello'

    # 6. How do you define optional properties in TypeScript interfaces?

    Answer: You define optional properties in TypeScript interfaces by adding a question mark (?) after the property name.

    Usage Example:

    interface Person {
  name: string;
  age?: number;
}

const person1: Person = { name: "John" }; // Valid
const person2: Person = { name: "Doe", age: 25 }; // Valid

    # 7. What is type assertion in TypeScript and how do you use it?

    Answer: Type assertion is a way to tell the TypeScript compiler to treat a value as a specific type. It doesn't perform any runtime type checking or conversion.

    Usage Example:

    let someValue: any = "Hello World";
let strLength: number = (someValue as string).length;

let someValue2: any = "Hello World";
let strLength2: number = (<string>someValue2).length;

    # 8. How do you handle null and undefined in TypeScript?

    Answer: TypeScript has special types null and undefined. By default, null and undefined are not included in types, but you can explicitly include them.

    Usage Example:

    let name: string | null = null;
let age: number | undefined = undefined;

    # 9. What are decorators in TypeScript?

    Answer: Decorators are a special kind of declaration that can be attached to a class, method, accessor, property, or parameter. They are used to add metadata or modify the behavior of the target they are applied to.

    Usage Example:

    function Log(target: any, key: string) {
  let value = target[key];

  const getter = () => {
    console.log(`Getting value: ${value}`);
    return value;
  };

  const setter = (newVal: any) => {
    console.log(`Setting value: ${newVal}`);
    value = newVal;
  };

  Object.defineProperty(target, key, {
    get: getter,
    set: setter,
    enumerable: true,
    configurable: true,
  });
}

class Person {
  @Log
  public name: string;

  constructor(name: string) {
    this.name = name;
  }
}

const person = new Person("John");
person.name = "Doe";
console.log(person.name);

    # 10. How do you define and use enums in TypeScript?

    Answer: Enums are a way to define a set of named constants in TypeScript.

    Usage Example:

    enum Direction {
  Up = 1,
  Down,
  Left,
  Right,
}

let direction: Direction = Direction.Up;

    # 11. What are union types in TypeScript?

    Answer: Union types allow you to define a variable that can hold more than one type.

    Usage Example:

    let value: number | string;
value = 10; // Valid
value = "Hello"; // Valid

    # 12. How do you define a function in TypeScript?

    Answer: You define a function in TypeScript with type annotations for the parameters and the return type.

    Usage Example:

    function add(a: number, b: number): number {
  return a + b;
}

    # 13. What is the unknown type in TypeScript?

    Answer: The unknown type represents a value that could be of any type. It is safer than any because you cannot perform operations on an unknown type without first narrowing it down to a specific type.

    Usage Example:

    let value: unknown = "Hello";

if (typeof value === "string") {
  console.log(value.toUpperCase()); // Valid
}

    # 14. What are type guards in TypeScript?

    Answer: Type guards are expressions that perform a runtime check to ensure a variable is of a specific type, allowing TypeScript to narrow down the type within a specific block of code.

    Usage Example:

    function isString(value: any): value is string {
  return typeof value === "string";
}

function example(value: string | number) {
  if (isString(value)) {
    console.log(value.toUpperCase()); // TypeScript knows value is a string here
  } else {
    console.log(value.toFixed(2)); // TypeScript knows value is a number here
  }
}

    # 15. How do you handle async/await in TypeScript?

    Answer: Async/await in TypeScript works the same way as in JavaScript, but with added type annotations.

    Usage Example:

    async function fetchData(url: string): Promise<any> {
  const response = await fetch(url);
  const data = await response.json();
  return data;
}

fetchData("https://api.example.com/data").then((data) => {
  console.log(data);
});

    # 16. What are type aliases in TypeScript and how do you use them?

    Answer: Type aliases in TypeScript are used to create a new name for a type. They can be used for primitives, union types, tuples, and any other type.

    Usage Example:

    type ID = string | number;

function printId(id: ID) {
  console.log(id);
}

printId(123); // Valid
printId("ABC"); // Valid

    # 17. How do you define a tuple type in TypeScript?

    Answer: A tuple type in TypeScript allows you to define an array with fixed types at specific positions.

    Usage Example:

    let tuple: [string, number];
tuple = ["Hello", 10]; // Valid

    # 18. What are the differences between interface and type in TypeScript?

    Answer:

    • Interfaces: Primarily used to define the shape of objects. They support declaration merging and can be implemented or extended.
    • Type Aliases: Used to create a new name for any type. They cannot be merged but can create more complex types like unions and intersections.

    Example:

    interface User {
  name: string;
  age: number;
}

type UserAlias = {
  name: string;
  age: number;
};

    # 19. How do you use the readonly modifier in TypeScript?

    Answer: The readonly modifier in TypeScript is used to make properties immutable after they are initialized.

    Usage Example:

    interface User {
  readonly id: number;
  name: string;
}

let user: User = { id: 1, name: "John" };
user.name = "Doe"; // Valid
user.id = 2; // Error: Cannot assign to 'id' because it is a read-only property

    # 20. How do you define a class in TypeScript?

    Answer: You define a class in TypeScript using the class keyword along with access modifiers, constructors, and methods.

    Usage Example:

    class Person {
  private name: string;

  constructor(name: string) {
    this.name = name;
  }

  getName(): string {
    return this.name;
  }
}

const person = new Person("John");
console.log(person.getName());

    # 21. What are access modifiers in TypeScript and how do you use them?

    Answer: Access modifiers in TypeScript control the visibility of class members. The three main access modifiers are public, private, and protected.

    • Public: Members are accessible from anywhere.
    • Private: Members are only accessible within the class.
    • Protected: Members are accessible within the class and its subclasses.

    Usage Example:

    class Person {
  public name: string;
  private age: number;
  protected gender: string;

  constructor(name: string, age: number, gender: string) {
    this.name = name;
    this.age = age;
    this.gender = gender;
  }
}

class Employee extends Person {
  constructor(name: string, age: number, gender: string) {
    super(name, age, gender);
    console.log(this.gender); // Valid
  }
}

const person = new Person("John", 30, "Male");
console.log(person.name); // Valid
console.log(person.age); // Error: 'age' is private
console.log(person.gender); // Error: 'gender' is protected

    # 22. How do you create an abstract class in TypeScript?

    Answer: An abstract class in TypeScript is a class that cannot be instantiated and is meant to be subclassed. It can contain abstract methods that must be implemented by subclasses.

    Usage Example:

    abstract class Animal {
  abstract makeSound(): void;

  move(): void {
    console.log("Moving...");
  }
}

class Dog extends Animal {
  makeSound(): void {
    console.log("Barking...");
  }
}

const dog = new Dog();
dog.makeSound(); // Output: Barking...
dog.move(); // Output: Moving...

    # 23. What are namespace in TypeScript and how do you use them?

    Answer: Namespaces in TypeScript are used to organize code into logical groups and to prevent name collisions.

    Usage Example:

    namespace Utils {
  export function add(a: number, b: number): number {
    return a + b;
  }

  export function subtract(a: number, b: number): number {
    return a - b;
  }
}

console.log(Utils.add(5, 3)); // Output: 8
console.log(Utils.subtract(5, 3)); // Output: 2

    # 24. How do you use type and interface together in TypeScript?

    Answer: You can use type and interface together by extending interfaces with types or vice versa.

    Usage Example:

    type Address = {
  street: string;
  city: string;
};

interface Person {
  name: string;
  age: number;
}

interface Employee extends Person, Address {
  position: string;
}

const employee: Employee = {
  name: "John",
  age: 30,
  street: "123 Main St",
  city: "New York",
  position: "Developer",
};

    # 25. How do you handle exceptions in TypeScript?

    Answer: Exception handling in TypeScript is done using try, catch, and finally blocks, similar to JavaScript.

    Usage Example:

    try {
  throw new Error("Something went wrong!");
} catch (error) {
  console.error(error.message);
} finally {
  console.log("Cleanup code goes here");
}

    # 26. What is type inference in TypeScript?

    Answer: Type inference in TypeScript is the ability of the compiler to automatically deduce the type of a variable based on its value.

    Usage Example:

    let x = 10; // TypeScript infers 'x' as number
x = "Hello"; // Error: Type 'string' is not assignable to type 'number'

    # 27. How do you declare an array in TypeScript?

    Answer: You can declare an array in TypeScript using two ways: with square brackets [] or using the Array generic type.

    Usage Example:

    let numbers: number[] = [1, 2, 3];
let strings: Array<string> = ["a", "b", "c"];

    # 28. How do you use never type in TypeScript?

    Answer: The never type in TypeScript represents the type of values that never occur. It is used for functions that never return, such as those that throw an error or have an infinite loop.

    Usage Example:

    function throwError(message: string): never {
  throw new Error(message);
}

function infiniteLoop(): never {
  while (true) {}
}

    # 29. What is type compatibility in TypeScript?

    Answer: Type compatibility in TypeScript is based on structural subtyping, which means that types are compatible if their members are compatible.

    Usage Example:

    interface Named {
  name: string;
}

let person: Named;
let employee = { name: "John", age: 30 };

person = employee; // Valid because 'employee' has a 'name' property

    # 30. How do you handle overloading in TypeScript?

    Answer: Function overloading in TypeScript allows you to define multiple signatures for a function. The actual implementation must handle all cases.

    Usage Example:

    function add(a: number, b: number): number;
function add(a: string, b: string): string;
function add(a: any, b: any): any {
  return a + b;
}

console.log(add(1, 2)); // Output: 3
console.log(add("Hello, ", "World!")); // Output: 'Hello, World!'

    # 31. How do you create a union of literal types in TypeScript?

    Answer: A union of literal types in TypeScript allows you to specify that a value can be one of a set of specific values.

    Usage Example:

    type Direction = "up" | "down" | "left" | "right";

function move(direction: Direction) {
  console.log(`Moving ${direction}`);
}

move("up"); // Valid
move("left"); // Valid
move("forward"); // Error: Argument of type '"forward"' is not assignable to parameter of type 'Direction'

    # 32. What is the Partial utility type in TypeScript?

    Answer: The Partial utility type constructs a type with all properties of the specified type set to optional.

    Usage Example:

    interface User {
  id: number;
  name: string;
  age: number;
}

const updateUser = (user: Partial<User>) => {
  // Update user logic
};

updateUser({ id: 1, name: "John" }); // Valid
updateUser({ age: 25 }); // Valid

    # 33. How do you define a mapped type in TypeScript?

    Answer: Mapped types in TypeScript allow you to create new types by transforming existing ones.

    Usage Example:

    type Readonly<T> = {
  readonly [P in keyof T]: T[P];
};

interface User {
  id: number;
  name: string;
}

const user: Readonly<User> = {
  id: 1,
  name: "John",
};

user.id = 2; // Error: Cannot assign to 'id' because it is a read-only property

    # 34. What are index signatures in TypeScript and how do you use them?

    Answer: Index signatures in TypeScript allow you to define types for properties that have not been explicitly defined in the type.

    Usage Example:

    interface StringArray {
  [index: number]: string;
}

let myArray: StringArray;
myArray = ["Bob", "Fred"];

let myStr: string = myArray[0]; // Valid

    # 35. What is the Record utility type in TypeScript?

    Answer: The Record utility type constructs a type with a set of properties K of type T.

    Usage Example:

    type Role = "admin" | "user";
interface User {
  id: number;
  name: string;
}

const userRoles: Record<Role, User> = {
  admin: { id: 1, name: "John" },
  user: { id: 2, name: "Doe" },
};

    # 36. How do you create a conditional type in TypeScript?

    Answer: Conditional types in TypeScript allow you to create types that depend on a condition.

    Usage Example:

    type IsString<T> = T extends string ? "string" : "not string";

type A = IsString<string>; // 'string'
type B = IsString<number>; // 'not string'

    # 37. What is the Exclude utility type in TypeScript?

    Answer: The Exclude utility type constructs a type by excluding from union T all types that are assignable to U.

    Usage Example:

    type T0 = Exclude<"a" | "b" | "c", "a">; // 'b' | 'c'
type T1 = Exclude<string | number | (() => void), Function>; // string | number

    # 38. What is the Extract utility type in TypeScript?

    Answer: The Extract utility type constructs a type by extracting from union T all types that are assignable to U.

    Usage Example:

    type T0 = Extract<"a" | "b" | "c", "a" | "f">; // 'a'
type T1 = Extract<string | number | (() => void), Function>; // () => void

    # 39. How do you use type predicates in TypeScript?

    Answer: Type predicates are used to narrow the type within a conditional block, allowing TypeScript to understand the type of a variable.

    Usage Example:

    function isString(value: any): value is string {
  return typeof value === "string";
}

function example(value: string | number) {
  if (isString(value)) {
    console.log(value.toUpperCase()); // TypeScript knows value is a string here
  } else {
    console.log(value.toFixed(2)); // TypeScript knows value is a number here
  }
}

    # 40. What are module augmentation and declaration merging in TypeScript?

    Answer:

    • Module Augmentation: Extends existing modules by adding new declarations.
    • Declaration Merging: Combines multiple declarations with the same name into a single definition.

    Usage Example:

    // Declaration merging
interface Person {
  name: string;
}

interface Person {
  age: number;
}

const person: Person = { name: "John", age: 30 }; // Valid

// Module augmentation
// existing-module.ts
export interface User {
  name: string;
}

// augment-module.ts
import { User } from "./existing-module";

declare module "./existing-module" {
  interface User {
    age: number;
  }
}

const user: User = { name: "John", age: 30 }; // Valid

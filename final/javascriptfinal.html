# JavaScript Interview Questions and Answers

# 1. What is the difference between var, let, and const?

Answer:

  • var declarations are function-scoped or globally scoped and can be redeclared. Variables declared with var are hoisted to the top of their scope.
  • let declarations are block-scoped and cannot be redeclared within the same scope. Variables declared with let are also hoisted but not initialized.
  • const declarations are block-scoped like let, but they must be initialized at the time of declaration and cannot be reassigned. The value it holds can be mutated if it is an object or array.
var x = 1;
let y = 2;
const z = 3;

# 2. What is hoisting in JavaScript?

Answer: Hoisting is a JavaScript mechanism where variables and function declarations are moved to the top of their containing scope during the compilation phase. This means you can use functions and variables before they are declared in the code.

console.log(a); // undefined
var a = 10;

hoistedFunction(); // Works because function declarations are hoisted
function hoistedFunction() {
  console.log("Hoisted!");
}

# 3. Explain closures in JavaScript.

Answer: A closure is a function that remembers its outer variables and can access them. In other words, a closure gives you access to an outer function's scope from an inner function.

function outerFunction() {
  let outerVariable = "I am outside!";

  function innerFunction() {
    console.log(outerVariable); // 'I am outside!'
  }

  return innerFunction;
}

const closure = outerFunction();
closure();

# 4. What are arrow functions, and how do they differ from regular functions?

Answer: Arrow functions are a concise way to write function expressions in ES6. They have a shorter syntax and do not have their own this, arguments, super, or new.target.

// Regular function
function add(a, b) {
  return a + b;
}

// Arrow function
const add = (a, b) => a + b;

# 5. What is the this keyword in JavaScript?

Answer: The this keyword refers to the object it belongs to. In a method, this refers to the owner object. Alone, this refers to the global object (in non-strict mode).

const obj = {
  name: "Alice",
  getName: function () {
    return this.name;
  },
};

console.log(obj.getName()); // 'Alice'

# 6. Explain the concept of promises and async/await.

Answer: Promises are used to handle asynchronous operations in JavaScript. They represent a value that may be available now, in the future, or never.

const promise = new Promise((resolve, reject) => {
  // some async operation
  if (success) {
    resolve("Success!");
  } else {
    reject("Error!");
  }
});

promise
  .then((result) => {
    console.log(result); // 'Success!'
  })
  .catch((error) => {
    console.log(error); // 'Error!'
  });

Async/await is syntactic sugar built on top of promises to write asynchronous code in a synchronous manner.

async function asyncFunction() {
  try {
    let result = await promise;
    console.log(result); // 'Success!'
  } catch (error) {
    console.log(error); // 'Error!'
  }
}

# 7. What is event delegation?

Answer: Event delegation is a technique of handling events efficiently by using a single event listener to manage all events of a particular type for child elements. It takes advantage of event bubbling.

document.querySelector("#parent").addEventListener("click", function (event) {
  if (event.target && event.target.matches("button.class-name")) {
    console.log("Button clicked!");
  }
});

# 8. What is the difference between == and ===?

Answer: == is the loose equality operator, which performs type coercion if the values are of different types before comparing. === is the strict equality operator, which does not perform type coercion and only returns true if both values and types are the same.

console.log(5 == "5"); // true
console.log(5 === "5"); // false

# 9. What is the purpose of the call, apply, and bind methods in JavaScript?

Answer:

  • call: Invokes a function with a specified this value and arguments provided one by one.
  • apply: Invokes a function with a specified this value and arguments provided as an array.
  • bind: Creates a new function that, when called, has its this keyword set to the provided value, with a given sequence of arguments preceding any provided when the new function is called.
function greet(greeting, name) {
  console.log(`${greeting}, ${name}`);
}

greet.call(null, "Hello", "Alice");
greet.apply(null, ["Hi", "Bob"]);
const greetAlice = greet.bind(null, "Hey", "Alice");
greetAlice(); // 'Hey, Alice'

# 10. What is a prototype in JavaScript?

Answer: Every JavaScript object has a prototype. A prototype is also an object. All JavaScript objects inherit their properties and methods from their prototype.

function Person(name) {
  this.name = name;
}

Person.prototype.greet = function () {
  console.log(`Hello, my name is ${this.name}`);
};

const alice = new Person("Alice");
alice.greet(); // 'Hello, my name is Alice'

# 11. What is the event loop in JavaScript?

Answer: The event loop is a fundamental concept that allows JavaScript to perform non-blocking operations, despite being single-threaded. It continuously checks the message queue and the call stack. If the call stack is empty, it takes the first event from the queue and pushes it onto the call stack, effectively running it.

# 12. What are higher-order functions?

Answer: Higher-order functions are functions that can take other functions as arguments, return functions, or both. They allow for function composition and abstraction.

function higherOrderFunction(callback) {
  return function (x) {
    return callback(x);
  };
}

const double = higherOrderFunction((x) => x * 2);
console.log(double(5)); // 10

# 13. Explain the concept of "callback hell."

Answer: Callback hell refers to the situation where multiple nested callbacks make the code hard to read and maintain. It often occurs in asynchronous programming.

firstFunction(function (result1) {
  secondFunction(result1, function (result2) {
    thirdFunction(result2, function (result3) {
      // and so on...
    });
  });
});

# 14. What are modules in JavaScript?

Answer: Modules are reusable pieces of code that can be exported from one script and imported into another. They help organize and structure code.

// myModule.js
export function sayHello() {
  console.log("Hello");
}

// main.js
import { sayHello } from "./myModule.js";
sayHello(); // 'Hello'

# 15. Explain the difference between synchronous and asynchronous code.

Answer:

  • Synchronous code is executed in sequence, blocking subsequent operations until the current one completes.

  • Asynchronous code allows multiple operations to run in parallel, enabling non-blocking behavior.

# 16. What is the spread operator and how is it used?

Answer: The spread operator (...) allows an iterable (e.g., array) to be expanded in places where zero or more arguments (for function calls) or elements (for array literals) are expected.

const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5]; // [1, 2, 3, 4, 5]

# 17. What is the purpose of the map method in JavaScript?

Answer:

The map method creates a new array populated with the results of calling a provided function on every element in the calling array.

const numbers = [1, 2, 3];
const doubled = numbers.map((x) => x * 2); // [2, 4, 6]

# 18. What is a promise chain?

Answer:

A promise chain is a series of .then methods chained together, allowing for the execution of asynchronous operations in sequence. Each .then returns a new promise.

doSomething()
  .then((result) => doSomethingElse(result))
  .then((newResult) => doThirdThing(newResult))
  .catch((error) => console.error(error));

# 19. What is destructuring in JavaScript?

Answer:

Destructuring is a syntax that allows you to unpack values from arrays or properties from objects into distinct variables.

// Array destructuring
const [a, b] = [1, 2]; // a = 1, b = 2

// Object destructuring
const { name, age } = { name: "Alice", age: 25 }; // name = 'Alice', age = 25

# 20. What is the purpose of the reduce method in JavaScript?

Answer: The reduce method executes a reducer function on each element of the array, resulting in a single output value. It's often used for summing up values or flattening arrays.

const sum = [1, 2, 3, 4].reduce((acc, val) => acc + val, 0); // 10

# 21. Explain async and await in JavaScript.

Answer: async and await are syntactic sugar built on top of promises to make asynchronous code look and behave more like synchronous code. async functions return a promise, and await pauses the execution of the async function until the promise is resolved.

async function fetchData() {
  const response = await fetch("https://api.example.com/data");
  const data = await response.json();
  return data;
}

# 22. What is memoization?

Answer: Memoization is an optimization technique used to speed up function execution by storing the results of expensive function calls and returning the cached result when the same inputs occur again.

function memoize(fn) {
  const cache = {};
  return function (...args) {
    const key = JSON.stringify(args);
    if (!cache[key]) {
      cache[key] = fn(...args);
    }
    return cache[key];
  };
}

const factorial = memoize(function (n) {
  if (n === 0) {
    return 1;
  }
  return n * factorial(n - 1);
});

console.log(factorial(5)); // 120

# 23. What is the difference between null and undefined?

Answer:

  • null is an assignment value that represents the intentional absence of any object value.

  • undefined means a variable has been declared but has not yet been assigned a value.

let a; // undefined
let b = null; // null

# 24. What is the difference between a function declaration and a function expression?

Answer:

  • Function Declaration: Defines a function with the specified parameters.
function sayHello() {
  console.log("Hello");
}
  • Function Expression: Defines a function inside an expression, and can be anonymous.
const sayHello = function () {
  console.log("Hello");
};

# 25. Explain the concept of immutability.

Answer: Immutability means that an object's state cannot be modified after it is created. Instead, new objects are created with the new state.

const obj = { name: "Alice" };
const newObj = { ...obj, age: 25 }; // Creates a new object

# 26. What are IIFEs (Immediately Invoked Function Expressions)?

Answer: IIFEs are functions that are executed immediately after they are defined. They are often used to create a new scope and avoid polluting the global namespace.

(function () {
  console.log("IIFE");
})();

# 27. What is the rest parameter and how is it used?

Answer: The rest parameter (...args) allows a function to accept an indefinite number of arguments as an array.

function sum(...args) {
  return args.reduce((acc, val) => acc + val, 0);
}

console.log(sum(1, 2, 3)); // 6

# 28. Explain the concept of "debouncing."

Answer: Debouncing is a programming practice used to ensure that a function is only executed after a specified amount of time has passed since it was last invoked. It's commonly used in handling events like resizing or scrolling.

function debounce(func, wait) {
  let timeout;
  return function (...args) {
    clearTimeout(timeout);
    timeout = setTimeout(() => func.apply(this, args), wait);
  };
}

const handleResize = debounce(() => {
  console.log("Resized");
}, 300);

window.addEventListener("resize", handleResize);

# 29. What is a generator function?

Answer: A generator function is a special type of function that can pause execution and resume it later. It is defined using the function* syntax and uses yield to pause execution

function* generateNumbers() {
  yield 1;
  yield 2;
  yield 3;
}

const generator = generateNumbers();
console.log(generator.next().value); // 1
console.log(generator.next().value); // 2
console.log(generator.next().value); // 3

# 30. What is the difference between deep copy and shallow copy?

Answer:

  • Shallow Copy: Copies the object's top-level properties but does not recursively copy nested objects. Changes to nested objects affect both copies.
const original = { a: 1, b: { c: 2 } };
const shallowCopy = { ...original };
shallowCopy.b.c = 3;
console.log(original.b.c); // 3
  • Deep Copy: Recursively copies all nested objects, creating a completely independent clone.
const original = { a: 1, b: { c: 2 } };
const deepCopy = JSON.parse(JSON.stringify(original));
deepCopy.b.c = 3;
console.log(original.b.c); // 2

# 31. What is the difference between map and forEach?

Answer:

  • map: Returns a new array with the results of applying a function to each element. It does not modify the original array.
  • forEach: Executes a function on each element of the array but does not return anything. It can mutate the original array.
const numbers = [1, 2, 3];

// map
const doubled = numbers.map((num) => num * 2); // [2, 4, 6]

// forEach
numbers.forEach((num, index, arr) => {
  arr[index] = num * 2;
});
console.log(numbers); // [2, 4, 6]

# 32. What are template literals in JavaScript?

Answer: Template literals are string literals that allow embedded expressions. They are enclosed by backticks (), and expressions are embedded using ${expression}.

const name = "Alice";
const greeting = `Hello, ${name}!`;
console.log(greeting); // 'Hello, Alice!'

# 33. What is the Object.create method?

Answer: The Object.create method creates a new object with the specified prototype object and properties. It is used to set up prototype chains without using constructors.

const person = {
  greet: function () {
    console.log(`Hello, my name is ${this.name}`);
  },
};

const alice = Object.create(person);
alice.name = "Alice";
alice.greet(); // 'Hello, my name is Alice'

# 34. Explain the concept of "throttling."

Answer: Throttling is a technique to limit the number of times a function can execute over a specified time period. It is commonly used to improve performance in situations like resizing or scrolling events.

function throttle(func, limit) {
  let inThrottle;
  return function (...args) {
    if (!inThrottle) {
      func.apply(this, args);
      inThrottle = true;
      setTimeout(() => (inThrottle = false), limit);
    }
  };
}

const handleScroll = throttle(() => {
  console.log("Scroll event");
}, 1000);

window.addEventListener("scroll", handleScroll);

# 36. What are symbols in JavaScript?

Answer: Symbols are a new primitive type introduced in ES6. They are unique and immutable identifiers used to create unique property keys.

const sym = Symbol("description");
const obj = {
  [sym]: "value",
};

console.log(obj[sym]); // 'value'

# 37. What is the Set object in JavaScript?

Answer: The Set object lets you store unique values of any type, whether primitive values or object references. It automatically removes duplicates.

const set = new Set([1, 2, 2, 3, 4]);
console.log(set); // Set { 1, 2, 3, 4 }
set.add(5);
console.log(set); // Set { 1, 2, 3, 4, 5 }

# 38. What is the WeakMap object in JavaScript?

Answer: The WeakMap object is a collection of key-value pairs where the keys are objects and the values can be arbitrary values. It does not prevent garbage collection of keys.

const weakMap = new WeakMap();
let obj = {};

weakMap.set(obj, "value");
console.log(weakMap.get(obj)); // 'value'

obj = null;
// Now, weakMap does not hold the reference to the key object, and it can be garbage collected

# 39. Explain the concept of "currying."

Answer: Currying is a technique of transforming a function that takes multiple arguments into a sequence of functions that each take a single argument.

function curry(func) {
  return function curried(...args) {
    if (args.length >= func.length) {
      return func.apply(this, args);
    } else {
      return function (...args2) {
        return curried.apply(this, args.concat(args2));
      };
    }
  };
}

function add(a, b, c) {
  return a + b + c;
}

const curriedAdd = curry(add);
console.log(curriedAdd(1)(2)(3)); // 6

# 40. What is the difference between Object.freeze and Object.seal?

Answer:

  • Object.freeze: Makes an object immutable. Properties cannot be added, removed, or changed.

  • Object.seal: Prevents adding or removing properties from an object, but existing properties can still be modified.

const obj = { a: 1 };
Object.freeze(obj);
obj.a = 2; // No effect
obj.b = 3; // No effect

const sealedObj = { a: 1 };
Object.seal(sealedObj);
sealedObj.a = 2; // Works
sealedObj.b = 3; // No effect

# 41. What is the purpose of the typeof operator?

Answer: The typeof operator returns a string indicating the type of the unevaluated operand.

console.log(typeof 42); // 'number'
console.log(typeof "Hello"); // 'string'
console.log(typeof true); // 'boolean'
console.log(typeof undefined); // 'undefined'
console.log(typeof null); // 'object' (this is a quirk in JavaScript)
console.log(typeof {}); // 'object'
console.log(typeof []); // 'object'
console.log(typeof function () {}); // 'function'

# 43. What is the Reflect API in JavaScript?

Answer: The Reflect API provides a set of static methods to perform common object operations. It is designed to complement the Proxy object.

const obj = { a: 1 };
Reflect.set(obj, "b", 2);
console.log(obj.b); // 2

# 44. What is the Intl object in JavaScript?

Answer: The Intl object is the namespace for the ECMAScript Internationalization API, which provides language-sensitive string comparison, number formatting, and date and time formatting.

const number = 123456.789;
console.log(new Intl.NumberFormat("en-US").format(number)); // '123,456.789'
console.log(new Intl.NumberFormat("de-DE").format(number)); // '123.456,789'

# 45. What is a polyfill in JavaScript?

Answer: A polyfill is a piece of code (usually JavaScript on the web) used to provide modern functionality on older browsers that do not natively support it.

if (!Array.prototype.includes) {
  Array.prototype.includes = function (element) {
    return this.indexOf(element) !== -1;
  };
}

# 46. Explain the Promise.all method.

Answer:

Promise.all takes an iterable of promises and returns a single promise that resolves when all of the input promises have resolved, or rejects if any of the input promises reject.

const promise1 = Promise.resolve(3);
const promise2 = 42;
const promise3 = new Promise((resolve, reject) => {
  setTimeout(resolve, 100, "foo");
});

Promise.all([promise1, promise2, promise3]).then((values) => {
  console.log(values); // [3, 42, "foo"]
});

# 47. What is the URL object in JavaScript?

Answer: The URL object provides a simple way to parse, construct, normalize, and encode URLs.

const url = new URL("https://example.com/path?name=abc&age=123");
console.log(url.hostname); // 'example.com'
console.log(url.pathname); // '/path'
console.log(url.searchParams.get("name")); // 'abc'

# 48. Explain the import and export statements in JavaScript.

Answer:

export: Used to export functions, objects, or primitives from a given file so they can be used in other programs with the import statement.

import: Used to import functions, objects, or primitives that have been exported from an external module.

// myModule.js
export const name = "Alice";
export function greet() {
  console.log("Hello");
}

// main.js
import { name, greet } from "./myModule.js";
console.log(name); // 'Alice'
greet(); // 'Hello'

# 49. What are the differences between synchronous and asynchronous iterations in JavaScript?

Answer:

  • Synchronous Iteration: Operations are executed sequentially. The next operation starts only when the current one completes.

  • Asynchronous Iteration: Operations are executed independently, allowing for non-blocking behavior using async/await or Promises.

# 50. What is eval in JavaScript and why is it considered harmful?

Answer: eval is a function that evaluates JavaScript code represented as a string. It is considered harmful because it can execute arbitrary code, leading to security vulnerabilities, and it can degrade performance by preventing certain optimizations.

const x = 10;
const y = 20;
const result = eval("x + y");
console.log(result); // 30

It's recommended to avoid using eval whenever possible due to the risks associated with it.

# 51. What is a WeakSet in JavaScript?

Answer: A WeakSet is similar to a Set, but it only holds weak references to its objects, meaning that if there are no other references to an object stored in the WeakSet, it can be garbage collected. It cannot contain primitive values and does not have size, clear, keys, values, or entries methods.

let obj = { key: "value" };
const weakSet = new WeakSet();
weakSet.add(obj);
console.log(weakSet.has(obj)); // true
obj = null;
// obj is now eligible for garbage collection, and weakSet will no longer contain it

# 52. What is the difference between slice and splice?

Answer:

  • slice: Returns a shallow copy of a portion of an array into a new array object selected from start to end (end not included). The original array is not modified.

  • splice: Changes the contents of an array by removing or replacing existing elements and/or adding new elements in place. It modifies the original array.

const arr = [1, 2, 3, 4, 5];

// slice
const slicedArr = arr.slice(1, 3); // [2, 3]
console.log(arr); // [1, 2, 3, 4, 5]

// splice
const splicedArr = arr.splice(1, 2); // [2, 3]
console.log(arr); // [1, 4, 5]

# 53. What is the purpose of the super keyword in JavaScript?

Answer: The super keyword is used to call the constructor of the parent class and to access its properties and methods. It is often used in subclass constructors to invoke the parent class's constructor.

class Parent {
  constructor(name) {
    this.name = name;
  }

  greet() {
    console.log(`Hello, my name is ${this.name}`);
  }
}

class Child extends Parent {
  constructor(name, age) {
    super(name);
    this.age = age;
  }

  greet() {
    super.greet();
    console.log(`I am ${this.age} years old`);
  }
}

const child = new Child("Alice", 10);
child.greet();
// Output:
// Hello, my name is Alice
// I am 10 years old

# 54. What is event propagation in JavaScript?

Answer: Event propagation is the order in which events are captured and handled in the DOM. It includes three phases:

  • Capturing phase: The event travels from the window down to the target element.

  • Target phase: The event reaches the target element.

  • Bubbling phase: The event bubbles up from the target element to the window.

document.getElementById("child").addEventListener(
  "click",
  function (event) {
    console.log("Child clicked");
  },
  true

# 17. Explain the concept of debouncing and throttling in JavaScript.

Answer: Both are techniques to control the rate at which a function is executed.

Debouncing: Ensures that a function is invoked only once after a specified period of inactivity.

function debounce(func, delay) {
  let timeout;
  return function (...args) {
    clearTimeout(timeout);
    timeout = setTimeout(() => func.apply(this, args), delay);
  };
}

window.addEventListener(
  "resize",
  debounce(() => {
    console.log("Resized");
  }, 500)
);
  • Throttling: Ensures that a function is invoked at most once in a specified period.
function throttle(func, limit) {
  let inThrottle;
  return function (...args) {
    if (!inThrottle) {
      func.apply(this, args);
      inThrottle = true;
      setTimeout(() => (inThrottle = false), limit);
    }
  };
}

window.addEventListener(
  "scroll",
  throttle(() => {
    console.log("Scrolled");
  }, 1000)
);

# 42. What is memoization in JavaScript?

Answer:

Memoization is an optimization technique used to speed up function calls by caching the results of expensive function calls and returning the cached result when the same inputs occur again.

function memoize(fn) {
  const cache = new Map();
  return function (...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) {
      return cache.get(key);
    }
    const result = fn(...args);
    cache.set(key, result);
    return result;
  };
}

const add = (a, b) => a + b;
const memoizedAdd = memoize(add);
console.log(memoizedAdd(1, 2)); // 3, calculated
console.log(memoizedAdd(1, 2)); // 3, cached

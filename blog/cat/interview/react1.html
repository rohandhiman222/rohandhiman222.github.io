                # State

                • State is like a variable Which is being watched by react application. if there are any changes to the state, react will make sure that they are re-rendered.

                The changes are updating in the DOM as well.

                remember this is kind of a setter method which allows you to change the state.

                # What is state?

                Directly taking a variable will not refresh/render it on a page but when takes as a state - then whatever changes are done to the state, those are automatically reflected on the user interface.

                • States basically contain the data which changes & it is the data which the user see

                • There might be different variables also which you can take but they don't affect the user interface, so we don't take those variable as state.

                In other words

                States are also variables which are actually displayed on the user interface so we take it with a specific syntax provided by react

                # how to use state?

                When it comes to declaring a state, we use the useState() hook.

                • useState() always returns one array with two elements

                State value, Setter function a function which allows you to change the state value.

                # Class

                • in versions before React 16.8, class was the only component which could use state in it.

                • after react 16.8 even the function component can have status.

                • so earlier there were terms like stateful component, which use to refer to class component and state.

                • stateful component referred as class component

                • Stateless component referred as function component.

                Important

                now there is just class and function component as both have status.

                • components defined as class still provide more features than a function component

                • the React.Component needs to be extended to create a class component.

                • render() is very essential component life cycle method.

                • render() is needed when you have a class so whatever and whenever states are updated, this method will be called.

                • render() is containing the JSX which is the user interface.

                • in the case of class component, we have a common method called setState() which is used to change the state all the time.
